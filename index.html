暖心使用手册

0/0
谢谢你选择 BYND

慢慢来，我们陪你把每一步走明白

哪怕第一次接触 AI 也完全没关系。这里会从“点哪里”开始说起，遇到报错也可以按对应方法一点点排查。

从你关心的部分开始按主题阅读，或输入关键词搜索
暂时没找到对应内容 可以换个说法试试，例如“401”“视觉模型”或“备份”。
第一次打开：从锁屏走到桌面打开方式、解锁、翻页与布局

把 BYND 想成一台住在网页里的小手机就好。浏览器打开最省心；添加到主屏幕后，看起来会更像独立 App。

宝宝级打开步骤

  1. 用浏览器打开：在手机或电脑打开 Chrome、Edge 或 Safari，把你购买后拿到的 BYND 地址粘贴到地址栏，再点“前往”或按回车。
  2. 想放到桌面：iPhone / iPad 可点 Safari 的“分享”→“添加到主屏幕”；Android 可点浏览器右上角菜单→“添加到主屏幕”或“安装应用”。不同浏览器的名字可能略有差别。
  3. 如果使用 APK：只从购买渠道提供的可信页面获取。系统询问安装来源时，可以先核对文件名和来源；不确定时继续使用网页版也很好。
  4. 进入桌面：看到锁屏后轻点画面。默认没有解锁密码；如果后来自己设置了密码，就输入自己的密码进入。
  5. 认识桌面：左右滑动空白处切换页面，底部圆点会告诉你当前在哪一页。顶部是状态栏，桌面上的每个图标都可以点开。

想整理成自己的样子

  1. 长按桌面空白处或项目，进入编辑状态；也可以打开“美化”后点“开始编辑”。
  2. 拖动图标、文件夹和组件到喜欢的位置，需要时再调整大小或移动页面。
  3. 确认顺眼后点“保存布局”。“恢复原始”会回到初始摆放，如果现在的布局很喜欢，可以先保存主题或导出一次数据。

小贴士：网页需要网络才能首次加载。偶尔看到旧界面时，轻轻刷新一次通常就能更新；已经保存的本地数据一般不会因为普通刷新消失。

API 与模型：给角色接上“大脑”概念、配置、视觉模型与完整报错对照

BYND 负责呈现手机、角色和聊天体验，真正生成回复的是你连接的 AI 服务。这个连接就叫 API。第一次看见这些词容易发懵，我们先用生活里的比喻认识它们。

API像一条电话线：BYND 把你的消息送给 AI，再把回答接回来。
服务商提供这条线路和 AI 能力的平台，通常会有自己的控制台、计费与使用规则。
Base URL像服务地址。它常以 https:// 开头，有些服务以 /v1 结尾，以服务商页面给出的格式为准。
API Key像只属于你的钥匙，用来确认账户和额度。它和登录密码一样适合私下保存。
模型像不同型号的大脑：速度、价格、文字能力和看图能力可能不同。

开始前，先准备这三样

  • 服务商控制台里创建的一枚 API Key。
  • 服务商文档或购买页面给出的 Base URL。
  • 一个可用模型名，或让 BYND 测试后读取模型列表。

关于费用：BYND 与 AI 服务商通常是两套独立服务。AI 请求是否计费、余额如何充值、模型单价多少，都以你选择的服务商页面为准。先用少量额度测试，会更安心。

跟着点：连接聊天 API

  1. 找到入口：回到桌面，打开“设置”→“API 连接”→“添加 API”。
  2. 写个名称：例如“我的聊天 API”。这个名字只方便自己辨认，不影响连接。
  3. 粘贴 Base URL：从服务商说明页完整复制。若对方已经给出 /v1，就原样保留；若没有，也不用凭感觉添加，以对方文档为准。
  4. 粘贴 API Key：留意开头和结尾是否多了空格，也可以重新复制一次。Key 不会显示完整字符，这是正常的隐私保护。
  5. 点“测试”：成功后通常会出现可选模型。若失败，先记下页面上的数字代码，再看下方“报错翻译”。
  6. 选择聊天模型：在模型列表里选一个文字模型。列表较多时，可先选服务商推荐的通用聊天模型。
  7. 保存连接：点“保存”。如果有多张 API 卡片，再到“默认用途”把这一张设为聊天用途。
  8. 发一句测试消息：进入任意角色聊天，发送“你好”，再点 AI 回复按钮。能收到回答，就说明线路接好了。

看图、画图和语音不是同一件事

视觉模型 / 多模态模型：能读懂你发送的照片或截图。若普通文字能回复、发图却失败，可以换成明确支持图片输入的聊天模型。

生图模型：负责从文字生成图片，常用于自拍、通话画面等。它不等于“能看图”的视觉模型，可能需要单独选择生图用途或单独配置服务。

语音服务：把文字念成声音，属于 TTS。可以按设置页对应入口配置 MiniMax、OpenAI、ElevenLabs 或本地 TTS，再用“测试语音”听一下效果。

报错翻译：看到数字时怎么做

提示它大多在说什么可以这样试
400请求格式、参数、模型名或上下文不合适。重新选择模型；核对 Base URL；减少过长的角色卡、世界书或聊天记录；再试一条短消息。
401没有识别到有效 Key，可能复制不全、过期或填错位置。回到服务商控制台复制新 Key，去掉首尾空格，确认 Key 与 Base URL 来自同一家服务。
403Key 可能有效，但账户、地区、模型或接口权限暂未开放。查看服务商的模型权限、余额和地区说明；换成账户已开通的模型，或联系对应服务商确认。
404地址、接口路径或模型名没有找到。逐字核对 Base URL 和模型名；避免重复写两次 /v1；按服务商提供的兼容接口地址填写。
408服务端等待请求太久，网络或上游响应可能偏慢。检查网络，稍等片刻重试；缩短消息与上下文；也可以换响应更快的模型。
429请求频率、并发或额度碰到服务商限制。歇几分钟再试，查看余额与限额；减少连续点击或自动请求，必要时换一个有额度的连接。
500服务商内部临时出错。稍后再试；若持续出现，可换同服务的其他模型或另一条 API，并查看服务商状态页。
502网关没有从上游 AI 收到正常回答。通常是服务链路短暂波动，稍后重试或切换节点、模型、服务连接。
503服务正在维护、过载或暂时不可用。给它一点恢复时间，再试一次；急用时可切换备用模型或备用连接。
504网关等上游太久，最终超时。缩短角色设定和聊天上下文，换更快模型；网络稳定后重试，或稍后再回来。

没有数字代码，也可以这样看

  • 连接超时 / 请求超时：常见于网络不稳、模型繁忙、上下文太长或图片太大。先发一句很短的纯文字；若短消息成功，再精简角色卡、世界书和近期聊天，或换快一点的模型。
  • 空回复 / 什么都没显示：可能是服务商返回空内容、内容过滤、模型协议不兼容，或输出额度太小。新开一段聊天发简单句测试，再换通用聊天模型；若仍为空，可在服务商控制台查看调用记录。
  • Failed to fetch / 网络错误：通常是地址不可达、证书、网络拦截或浏览器 CORS 跨域限制。核对 https 地址并换网络测试；服务商若要求代理，可使用自己信任的后端或 Worker。
  • 模型列表为空：测试可能还没成功，或者服务商不开放模型列表。先确认连接测试通过；若页面允许手填模型名，可复制服务商文档中的准确名称。
  • 文字能聊，图片看不懂：当前模型可能只支持文字。换成服务商明确标注“Vision / 视觉 / 多模态 / 图片输入”的聊天模型，并适当压缩图片后再试。

隐私小提醒：聊天内容会经由你填写的服务商或代理发送给 AI。API Key 更适合只保存在自己的设备和可信页面里；截图求助时，可以先遮住 Key、完整地址和个人聊天内容。

预设、世界书与正则先怎么区分决定表达方式、补充设定与整理显示
预设像聊天的总导演，影响回复结构、语气和思考方式。
世界书像设定资料库，在关键词出现时把相关背景交给 AI。
正则像显示滤镜，可查找、隐藏或替换消息里的特定文字。
  1. 打开桌面底部“预设”，点“手动新建预设”填写名称和内容，再保存。
  2. 已有兼容 JSON 时，点“导入预设 (.json)”选择文件；导入后先读一遍内容，确认符合自己的聊天期待再启用。
  3. 预设不会代替角色卡或世界书。效果太强时，可以先关闭预设做一次对比，再决定是否调整。

温柔建议：刚开始用时先保持简单：一个角色、一份预设、少量世界书。确认聊天稳定后再慢慢叠加，更容易找到问题来自哪里。

领养角色并开始第一次聊天导入角色卡、发送消息与聊天进阶

导入一张角色卡

  1. 打开“微信”，点右上角“+”→“导入角色”。
  2. 手里有角色卡图片时，选“导入角色卡 (PNG)”并挑选文件。角色卡看起来是图片，但里面可能还带着名字、人设和开场白。
  3. 没有角色卡时，选“手动新建角色”，填写名字、简介和开场白，再确认添加。
  4. 回到消息或通讯录，点新角色的头像进入聊天。

发出第一条消息

  1. 在底部输入框写一句话，点“发送”。
  2. 想让角色回答时，点输入栏旁的 AI 回复按钮。若没有反应，可以回到上一章测试 API。
  3. 表情按钮可打开贴纸；“+”或更多工具里可以发图片、拍照、红包、转账或发起通话，实际入口会随屏幕宽度略有变化。
  4. 长按消息可查看复制、引用、编辑、撤回、删除或重新生成等操作。不同类型消息能看到的选项可能不同。

让 TA 更像你熟悉的人

聊天页右上角“更多”可以进入角色聊天设置，调整彼此称呼、人设、线上 / 线下场景、时间模式、世界书、头像和通话画面。改好后点页面底部“保存设置”，新设置会从后续聊天开始发挥作用。

群聊、朋友圈与丰富互动多人聊天、动态、通话和一起听
  1. 建群：微信右上角“+”→“发起群聊”，选择至少 2 位已有角色，填写群名后点“创建群聊”。群里的每次 AI 回复都可能消耗 API 额度。
  2. 朋友圈：打开“发现”→“朋友圈”。右上角相机可发布文字或图片动态；角色也可能评论。动态可收藏、转发、编辑或删除。
  3. AI 朋友圈：在角色设置里编辑 TA 的签名、背景和动态，“AI 写”会调用当前聊天 API 生成草稿，你仍可以改到满意再添加。
  4. 通话：聊天页“更多”里可选语音或视频通话。真实麦克风、相机和屏幕共享会先由浏览器询问，按自己的需要决定是否开启。
  5. 一起听：音乐详情或支持的聊天入口可以邀请角色一起听，歌词与评论依赖对应音乐来源。
  6. 看看 TA 的手机：双击角色头像进入char手机内容

更舒服的使用方式：发照片、开启相机或共享屏幕前，可以先扫一眼画面里有没有姓名、地址、二维码、通知内容等个人信息。

世界书与正则脚本给故事补背景，让消息显示更清爽

世界书：把设定按主题收好

  1. 打开“世界书”，点右上角“+”新建一本书，起一个自己看得懂的名字。
  2. 进入书本后新增条目，填写关键词和内容。例如关键词写“学校”，内容写学校名称、地点与规则。
  3. 保存条目后，到角色聊天设置里勾选这本世界书。关键词在聊天中出现时，相关内容更容易被带给 AI。
  4. 设定很多时可以拆成多条短内容。过长世界书会占用上下文，也可能让回复变慢。

正则：先从一条小规则试起

  1. 打开“正则脚本”，选择“全局通用”或“角色专属”，再点右上角“+”。
  2. 填写脚本名称、Regex（查找规则）和 Replacement（替换内容），保存后回聊天观察显示效果。
  3. 如果文字突然不见，可以先关闭最近添加的脚本。复杂规则更适合先在角色专属范围测试，满意后再考虑全局使用。
美化、主题与桌面组件把锁屏和桌面装扮成喜欢的样子
  1. 打开“美化”，在“全局壁纸设置”分别选择锁屏和桌面背景，也能开关锁屏页与状态栏。
  2. 继续往下看，可以调整锁屏拍立得、时间颜色、触碰特效、日历配色、字体、快捷按钮和 App 图标。
  3. 想移动桌面项目时，点“开始编辑”，拖到喜欢的位置并调整大小，再点“保存布局”。
  4. 全部搭配好后点顶部“保存”。想留下整套装扮，可进入“主题库”，命名后点“保存当前主题”。

显示小建议：浅壁纸配深色文字、深壁纸配浅色文字通常更清楚。高分辨率壁纸、自定义字体和大量图标会多占一些本地空间，设备变慢时可以从较轻的素材开始。

音乐与网易云歌单搜索、播放、搬入歌单和一起听
  1. 从桌面底部打开“音乐”，在搜索框输入歌名或歌手，可按页面提供的选项切换“多源”“网易云”或“公开”来源。
  2. 想同步网易云时，点“网易云音乐”入口，跟随页面扫码登录并读取歌单。
  3. 也可以打开“更多设置”，粘贴网易云歌单链接或 ID，再点“导入到音乐”。
  4. 点歌曲进入详情页查看歌词或评论；出现“一起听”时，可以选择一位角色陪伴。

加载不出来时：音乐播放和歌单解析依赖网络及第三方来源，偶尔会有地区、版权或接口波动。换一个来源、换首歌或稍后再试，通常比反复点击更轻松。

学习、记账、游戏与其他 App从实用小工具到陪伴玩法
学习填写学习内容、解释和例句，点“保存学习卡”；已有卡片可以复习或用于语言翻牌。
记账填写收支后点“+ 记一笔”，在月度区域看汇总。这里适合个人记录，金额仍以实际账单为准。
Game可体验狼人杀、肥鸡大冒险、跳一跳、倒水排序、2048、五子棋等。角色玩法通常需要角色与聊天 API。
盗梦空间选择入梦角色，点“生成梦境”。梦境文字由 AI 生成，因此会使用 API。
一起出门选择陪伴角色并开启定位，移动后可触发互动，也能拍照给 TA 看。定位精度取决于设备和浏览器。
PageMate在书架导入支持的文档或添加书籍链接，进入阅读器后可调字体和外观，并邀请角色一起读。

找不到某个 App：先左右滑动看看第二页，也可以检查桌面文件夹。若自己改过布局，进入美化编辑模式更容易找到被移动的图标。

监控、桌宠与屏幕陪伴让角色参与更多屏幕内场景
  1. 在“监控”中选择监控者接入 BYND 内部聊天，可查看剧情、后台提示和桌宠气泡，并按喜好调整视角与弹幕速度。
  2. 想用桌宠时，在监控工具里开启桌宠并绑定已接入角色。点击桌宠可以触发一次观察。
  3. 开启屏幕观察时，浏览器会弹出共享选择。只选自己愿意展示的窗口或标签页，用完点系统的“停止共享”即可。

隐私小提醒：共享画面里可能带有聊天通知、账号、文件名或工作内容。更安心的做法是先关掉无关窗口和通知，再只共享需要的单个标签页。

一起出门与 PageMate 共读定位陪伴、拍照互动和阅读

一起出门

  1. 打开“一起出门”，先选择“出门对象”，再点“开启”。
  2. 浏览器询问定位时，按自己的使用场景选择。开启后页面会记录距离，约每移动 500 米可叫醒角色互动。
  3. 点“拍照给他看”会打开相机或文件选择；照片能否被理解，还取决于聊天模型是否支持视觉输入。

PageMate 共读

  1. 在“书架”导入页面支持的本地文档，或添加书籍链接；“发现”里还可以按书名、作者或 ISBN 搜索。
  2. 选择书进入阅读器，用“字体”和“设置”调成舒服的阅读外观。
  3. 在“一起读”选择角色，在“想法”里查看阅读记录与互动。较大的 PDF 或复杂网页可能需要多等一会儿，也可能受来源限制。
相册、相机、权限与主动提醒保存图片，管理设备权限和后台消息
  1. 相册：打开第二页“相册”，按角色筛选聊天中保存的生成图。这里为空时，可以先在角色聊天里完成一次图片生成。
  2. 相机：点底部“相机”后，设备可能打开真实相机，也可能显示文件选择器，具体取决于浏览器能力和权限。
  3. 主动提醒：打开“设置”→“主动消息提醒”，开启“后台消息”并选择时间，再按系统弹窗决定是否允许通知。
  4. 改回权限：之前点了拒绝也没关系，可以到浏览器地址栏旁的网站设置，重新调整相机、麦克风、定位或通知。

关于提醒时间：浏览器为了省电可能暂停后台网页，所以提醒偶尔会晚一些。把 BYND 添加到主屏幕、减少系统省电限制，通常会更稳定，但仍会受设备策略影响。

备份、换设备与隐私温柔照看角色、聊天和自己的信息

角色、聊天、主题和部分进度主要保存在当前浏览器里。刷新页面通常没事，但清理站点数据、浏览器故障、卸载应用或更换设备，都可能让本地内容消失。定期导出一份，就像给这台小手机留一把备用钥匙。

三分钟做一次备份

  1. 打开“设置”→“数据管理”。
  2. 点“导出数据”，等待浏览器下载一个 JSON 文件。
  3. 把文件改成容易认的名字，例如“BYND-2026-07-11.json”。
  4. 保存到自己的加密网盘、加密磁盘或另一个可靠设备。保留最近两到三份，更容易回到合适的版本。

换手机或恢复数据

  1. 先在旧设备导出最新数据，再把 JSON 文件传到新设备。
  2. 新设备打开 BYND→“设置”→“数据管理”→“导入数据”，选择刚才的 JSON。
  3. 导入完成后，查看角色、聊天、主题和 API 卡片是否齐全，再发一句短消息测试。
  4. 如果导入前新设备已经有内容，可以先给新设备也导出一份，方便需要时回退。

这份备份里可能有什么

备份可能包含角色资料、聊天内容、图片记录、主题、定位相关记录以及 API 配置。它更适合放在只有自己能访问的位置；分享截图或把文件发给他人前，可以先想想里面是否带有私密内容。

更安心的顺序：遇到存储异常时，先导出备份，再使用“清理无效缓存”。浏览器的“清除网站数据”范围更大，可能连角色和聊天一起清掉，所以把它留作最后一步会更稳妥。

常见问题：按现象慢慢排查无回复、看不懂图片、权限与数据恢复
  1. AI 完全没有回复先确认网络能打开其他网页→到设置测试 API→确认聊天默认用途和模型已经保存→发一句短纯文字。若出现数字代码，可回到“API 与模型”对照表查看。
  2. 一直转圈，最后超时先停一下连续点击→换稳定网络→缩短消息→暂时关闭较大的世界书或预设→换更快模型。图片消息还可以先压缩尺寸。
  3. 收到空白气泡或空回复新开聊天发“你好”测试；换通用模型；检查服务商是否记录了内容过滤、输出上限或余额问题。只有某个角色为空时，可暂时停用该角色的正则和预设做对比。
  4. TA 看不懂照片确认聊天模型支持视觉 / 多模态输入，而不只是生图；换一张较小的 JPG 或 PNG，并在图片旁补一句“说说你看到了什么”。
  5. 生成不了图片确认已经配置生图模型，并在“默认用途”中选好图片用途。聊天视觉模型和生图模型是两类能力，只有其中一个时，另一项可能仍不可用。
  6. 模型列表一直为空重新测试 Base URL 与 Key；查看服务商是否支持读取模型列表。若支持手填，就从服务商文档复制准确模型名，留意大小写和连字符。
  7. 相机、麦克风、定位或通知打不开点浏览器地址栏旁的锁形图标或网站设置，找到对应权限并改为允许，再重新打开功能。系统设置里也可能有一层 App 权限。
  8. 图片、音乐或书籍加载失败这类内容还依赖第三方来源，换网络、换来源或稍后重试即可。只有某一个链接失败时，往往是来源本身不再可用。
  9. 换设备后内容不见了数据不会自动跟随账号同步。若旧设备仍能打开，从旧设备导出 JSON，再到新设备导入；没有备份时，可以检查旧浏览器是否仍保留站点数据。
  10. 页面变慢或空间不足先导出备份，再清理无效缓存；适当减少大图、壁纸、自定义字体和过长聊天。普通刷新可以先试，清除整个网站数据则更适合放在最后。

求助时带上这些信息会更快：设备和浏览器、使用的模型名、完整报错代码、问题发生在哪一步。API Key、私聊内容和完整备份文件可以先遮挡或省略。

生图如果提示 Failed to fetch，通常是这个站点不允许浏览器直连；请把这里换成你自己的 Worker/后端代理地址。
如果这个站点只有聊天模型，就再添加一个生图 API，并在上方“默认用途”里把它设成生图备用。
MiniMax 语音 API
接入方法 默认会请求 MiniMax Base URL + /v1/t2a_v2。 先在 MiniMax 官方创建或克隆音色，拿到 voice_id 后填到音色 ID。浏览器直连如果被 CORS 拦住，就把 Endpoint 换成你自己的 Worker/后端代理。 优势：中文、情绪和克隆音色更适合角色语音，按字符计费更好估成本。劣势：需要官方 Key，浏览器直连可能要代理。
OpenAI TTS API
接入方法 默认会请求 Base URL + /audio/speech，音色填 OpenAI 支持的内置 voice。 优势：稳定、通用音色质量好，适合开箱即用。劣势：ChatGPT 里的 Cove 不是公开 TTS API 音色，不适合做“上传声音模仿”的主方案；浏览器直连被 CORS 拦时要走代理。
ElevenLabs API
接入方法 默认会请求 Base URL + /v1/text-to-speech/{Voice ID}，Voice ID 从 ElevenLabs 的声音库或克隆声音里复制。 优势：音质和克隆生态成熟，适合高级音色。劣势：免费额度小，商业使用要看套餐，成本通常比 MiniMax 更高；浏览器直连被 CORS 拦时要走代理。
本地 TTS HTTP API
接入方法 Endpoint 填电脑上运行的 TTS HTTP 地址，接口需要允许浏览器跨域请求。 请求体会发送 text、model、voice、voice_id、format；返回 audio/*、audio_base64 或 audio_url 都能识别。 优势：成本最低、数据在自己电脑。劣势：离开家里要隧道/VPN/公网服务，手机和 GitHub Pages 测试会受 HTTPS 与局域网限制。
