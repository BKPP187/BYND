添加 API
生图如果提示 Failed to fetch，通常是这个站点不允许浏览器直连；请把这里换成你自己的 Worker/后端代理地址。
如果这个站点只有聊天模型，就再添加一个生图 API，并在上方“默认用途”里把它设成生图备用。
MiniMax 语音 API
接入方法 默认会请求 MiniMax Base URL + /v1/t2a_v2。 先在 MiniMax 官方创建或克隆音色，拿到 voice_id 后填到音色 ID。浏览器直连如果被 CORS 拦住，就把 Endpoint 换成你自己的 Worker/后端代理。 优势：中文、情绪和克隆音色更适合角色语音，按字符计费更好估成本。劣势：需要官方 Key，浏览器直连可能要代理。
OpenAI TTS API
接入方法 默认会请求 Base URL + /audio/speech，音色填 OpenAI 支持的内置 voice。 优势：稳定、通用音色质量好，适合开箱即用。劣势：ChatGPT 里的 Cove 不是公开 TTS API 音色，不适合做“上传声音模仿”的主方案；浏览器直连被 CORS 拦时要走代理。
ElevenLabs API
接入方法 默认会请求 Base URL + /v1/text-to-speech/{Voice ID}，Voice ID 从 ElevenLabs 的声音库或克隆声音里复制。 优势：音质和克隆生态成熟，适合高级音色。劣势：免费额度小，商业使用要看套餐，成本通常比 MiniMax 更高；浏览器直连被 CORS 拦时要走代理。
本地 TTS HTTP API
接入方法 Endpoint 填电脑上运行的 TTS HTTP 地址，接口需要允许浏览器跨域请求。 请求体会发送 text、model、voice、voice_id、format；返回 audio/*、audio_base64 或 audio_url 都能识别。 优势：成本最低、数据在自己电脑。劣势：离开家里要隧道/VPN/公网服务，手机和 GitHub Pages 测试会受 HTTPS 与局域网限制。
